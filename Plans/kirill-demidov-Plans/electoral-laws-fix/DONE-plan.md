# Plan: electoral-laws-fix

Цель: исправить три бага первого прогона `electoral_laws.py` и сфокусировать
сбор на законах об избирательной системе (метод распределения мест, порог,
тип системы). Данные нужны как справочник для объяснения расхождений между
фактом и пятью методами расчёта в electoral-calc.org.

Включаем все страны и регионы (Каталония, Шотландия и т.п.).

## 1. Исправить GLOBALCIT (0 законов)

**What:** Сайт грузит таблицу через DataTables/JS — статический HTML пустой.
Нужно найти источник данных.

**How:** В три попытки, первая успешная побеждает:
1. Парсить `<script>` теги страницы — DataTables часто встраивает данные как
   JS-массив (`var tableData = [...]` или `initDataTable([...])`). Извлечь
   через regex.
2. Найти в HTML `"ajax":` или `"url":` в DataTables конфиге → сделать
   `fetch()` на этот URL.
3. Fallback: попробовать WP REST API
   (`/wp-json/wp/v2/posts?per_page=100&categories=...`) — GLOBALCIT на
   WordPress, законы могут быть отдельным post type.

Если все три не дали данных — логировать предупреждение и продолжать без
GLOBALCIT.

## 2. Исправить ACE (только 29 законов)

**What:** Функция `_find_section` не находит нужную секцию на большинстве
страниц, и поиск заканчивается без результата.

**How:**
- Расширить ключевые слова секций: добавить `"electoral system"`,
  `"election code"`, `"constitutional"`, `"statutes"`.
- Если секция всё равно не найдена: сканировать **все** ссылки на странице
  через `_looks_like_law` (не ограничиваться разделом).
- Распечатывать `[no section, full scan]` чтобы было видно в логах.

## 3. Исправить IFES («Cookie setting» и шум)

**What:** Парсер принимает навигационные ссылки за страны.

**How:**
- Перед `upsert_country` проверять: название длиннее 2 символов, не входит
  в стоп-лист (`{"Cookie setting", "Home", "Back", "Next", "Search", "Login",
  "Menu", "Contact", "About", "Privacy", "Terms"}`).
- Стоп-лист хранить как `_SKIP_NAMES: frozenset[str]` в модуле.

## 4. Сфокусировать `_looks_like_law` на избирательной системе

**What:** Сейчас фильтр слишком широкий — тянет любые выборные документы
(финансирование кампаний, регистрация избирателей и т.п.).

**How:** Разделить на два уровня:
- **primary** — точные совпадения: `"electoral system"`, `"election code"`,
  `"electoral law"`, `"parliament act"`, `"proportional"`, `"seat allocation"`,
  `"constituency"`, `"кодекс"`, `"избирательный закон"`.
- **secondary** — нынешние широкие ключевые слова (`"law"`, `"act"`,
  `"election"`, `".pdf"`).

Функция принимает параметр `strict: bool = False`. При `strict=True`
использует только primary-список. В `_ace_scrape_country` и
`_ifes_scrape_country` передавать `strict=False` (не хотим потерять
покрытие), но при наличии секции — `strict=True`. Логировать уровень.

**What:** Различать законы об избирательной системе от общих выборных

**How:**

  EXISTS + `init_db` миграция).
- Заполнять `"electoral_system"` если совпадение по primary-списку, иначе
  `"general"`.
- Обновить `export_csv` — включить колонку в CSV.

## Зависимости и риски

- GLOBALCIT JS-структура может измениться — три попытки делают решение
  устойчивым.
- Строгий фильтр в `strict=True` может пропустить нестандартно названные
  законы — поэтому `strict=False` остаётся дефолтом вне секций.
- `ALTER TABLE` безопасен с `IF NOT EXISTS` — повторный запуск не сломает
  существующую БД.
